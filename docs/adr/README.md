# ADR (Architecture Decision Records)

## Objetivo

Esta pasta registra todas as decisões arquiteturais tomadas durante o desenvolvimento do Supervisor AI.

Uma ADR representa uma decisão importante que influencia a arquitetura do sistema.

---

# Quando criar uma ADR

Uma ADR deve ser criada sempre que houver uma decisão que possa impactar o projeto no futuro.

Exemplos:

- escolha de tecnologias;
- padrões arquiteturais;
- organização dos dados;
- integrações;
- decisões de segurança.

---

# Estrutura

Cada ADR deve conter:

- Contexto
- Problema
- Decisão
- Consequências
- Status

---

# Objetivo

As ADRs evitam que decisões importantes sejam perdidas ao longo da evolução do projeto.

Antes de alterar uma decisão arquitetural existente, deve-se consultar a ADR correspondente.
